Make a single line IDE terminal based flashcard syntax checker
(super easy GitHub project idea - at least it seems easy in 
my head when thinking out how it should work).

-System.out.println to give prompt in terminal.
-Use scanner to take user input (answer choice).
-format user input to take out white space so if user types 
slightly differently from reference answer choice it'll still 
accept as correct answer (as long as the only difference it
in spaces between certain characters/symbols). 
-If user inputs wrong answer 3+ times then they give failure
text and the correct answer is shown. Given prompt to type 
"retry", "quit", or "continue". Any other input will result
in the wrong answer prompt above being shown again (+ `their input`
"was not a valid option." + failure text). This also gives 
them time to read the reference answer & understand it.
    - "retry" : clears terminal and asks the question again.
    - "quit" : ends the program + output text "you have prematurely
    ended the study session".
    - "continue" : puts the failed question at the back of the
    questions queue for that deck. 

- Question type start simple, only a couple queues (referred to
as question "decks" in the program) of related question prompts
where the answer choice is a single line of code | this 
essentially functions as a syntax checker. Better than 
flashcards because you have to more precisely enter in the 
correct answer choice, compared to being able to roughly 
guess the answer and give yourself leeway on whether you 
got it right or not. 

- ✅️What data structure should the questions and answers be 
stored as?? : 
    - It all gets stored in a SequencedMap (Specifically a LinkedHashMap)
    
- ✅️What data structure should the decks holding the questions
be?? : 
    - After spending some time reviewing data structures I changed my decision 
    on using an ArrayDeque to using a Map based data structure for storing
    the questions and answers. At first I was going to use a TreeMap to be 
    able to have mapped questions and answers in a specific order that I
    could manipulate later. However, after I had spent some time trying 
    to think of ways to change the order of questions for removing and 
    adding missed questions to the back (incuding possibly creating a
    second TreeMap object to store missed questions while removing correctly 
    answered questions altogether, and then adding those missed questions back
    into the main question TreeMap: which seemed like a lot of code to have
    to read through for another person potentially having to review any of 
    my code in a real world scenario) I remembered reading documentation 
    about a specific type of Map called a SequencedMap during my time 
    reviewing data structures. I had to look up what it does (because I didn't 
    write down anything but the name upon first learning about it) and the 
    description seemed like it is a perfect fit for my specific use case. 

- Limit scope (what ideas do you have that you should not
focus on doing in this scale of a project?) :
    - NO user input adding to decks (would require adding a 
    database, and more time spent on the terminal prompt tree).
    - NO saving/tracking answers or progress (would require
    adding a database).
    - MAYBE add random image generator as "sticker" reward for
